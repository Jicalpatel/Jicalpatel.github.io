Distributed HIP DNN Pipeline

A scalable architecture that connects MPI4Py data producers with AMD HIP compute workers for distributed deep learning experiments.

Project Story

The system is designed around a producer-consumer flow: MPI ranks prepare and shard data, HIP workers receive mini-batches, GPU kernels run forward and backward DNN layer operations, and an orchestrator coordinates synchronization and metrics.

The architecture emphasizes practical systems thinking: batch streaming, wire formats, backpressure, GPU utilization, throughput, latency, and fair comparison against PyTorch DDP.

What makes it strong

  • Combines distributed Python orchestration with lower-level GPU kernel execution.
  • Tracks performance metrics such as samples per second, end-to-end latency, and scale efficiency.
  • Designed with pluggable kernels so optimized HIP implementations can replace baseline layers.
